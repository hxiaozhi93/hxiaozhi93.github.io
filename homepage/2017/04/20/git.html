(1)git基本内容与命令：

Windows：安装msysgit,打开git bash进行设置

git config --global user.name “your name” ，git config --global user.email "your email"

查看：git config user.name以及git config user.email

创建一个目录如gittest，然后把这个目录变成仓库，git init

清空屏幕：clear

输入ls -ah 可看到目录下多了.git，open .git,

在目录下，编写一个readme.txt 文件

把文件添加到仓库，git add readme.txt

把文件提交到仓库，git commit -m "write a readme.txt"

查看当前仓库状态：git status

查看修改状态：git diff，查看staged状态后：git diff --cached;修改了两次两个状态：git diff HEAD;

查看历史记录：git log,git log --oneline --graph

查看并简洁显示：git log --pretty=oneline

记录每一次命令：git reflog

add后回到add前：git reset 1.py，即从staged状态回到unstaged（modified）状态

回退到上一个版本：git reset --hard HEAD^或者回退多个版本：HEAD~100

回到过去版本：git reset --hard 版本id号

回到未来版本：先git reflog找到未来版本id号，再git reset --hard 版本id号

reset针对整个版本库，在commit之间穿梭，checkout针对单个文件，不同在于自身改变而朋友们没变

续:git checkout 过去id号 -- 1.py

丢弃工作区的修改：git checkout --file

撤销暂存区修改：git reset HEAD file

别名： git config --global alias.co checkout

查看配置： git config -l

暂存当前没有提交的代码： git stash

还原之前的代码： git stash apply

删除stash记录： git stash drop/pop/clear

(2)分支

创建分支a: git branch a

查看分支： git branch

切换到分支a：git checkout a

把a分支内容合并到master： 先切回到master，然后 git merge a

续：git merge --no--ff -m "keep merge info" dev

删除a分支： git branch -d a

打标签： git tag name

查看标签： git tag

查看标签信息：git show tagname

分支冲突：手动解决或者rebase

(3)远程仓库

GitHub：提供Git仓库托管服务

创建SSH Key：ssh-keygen -t rsa -C "your email",然后一路回车

生成.ssh文件，里面有id-rsa和id_rsa.pub

登录GitHub，选择 Add SSH key，添加id_rsa.pub中的内容

添加远程库：git remote add origin git@github.com:hxiaozhi93/hello-world

git push -u origin master

(4)从自己的github上clone内容到本地，修改后上传回到github：

首先在本地建立文件夹，如f://myclone

再打开Git Bash，输入cd f://myclone

再输入自己的github地址，如git clone https://github.com/hxiaozhi93/hello-world

再输入cd hello-world,接着修改hello-world文件内容

再输入git add .

再输入git commit -m "change hello-world"

再输入git push origin master

查看github上的hello-world文件内容，可发现内容已经修改
